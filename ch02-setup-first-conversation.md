## 2.1 安裝 VS Code + Claude Code Extension

三個步驟，5 分鐘搞定。

1. **安裝 VS Code** — 前往 [code.visualstudio.com](https://code.visualstudio.com) 下載安裝
2. **安裝 Extension** — 按 `Cmd+Shift+X`（Mac）/ `Ctrl+Shift+X`（Windows）開啟 Extensions 面板，搜尋 `Claude Code`，找 **by Anthropic** 的官方版本，點 Install，重啟 VS Code
3. **確認成功** — 重啟後左側活動列底部會出現 Claude 的對話泡泡圖示；按 **Cmd+Esc** 快速開關 Claude Panel

點圖示後會出現登入提示，用 Anthropic 帳號登入即可。

> 截圖教學之後會補上。

## 2.3 模型選擇指南

在 Claude Code 裡，你可以隨時切換不同的模型。按 `Cmd+P`（Mac）/ `Meta+P`（Linux/Windows）快速開啟模型選擇器。

### 三個模型的定位

| 模型 | 速度 | 能力 | 最適合 |
|------|------|------|--------|
| **Opus 4.6** | 慢 | 最強 | 複雜架構設計、大型重構、困難 bug |
| **Sonnet 4.6** | 快 | 強 | 日常 coding、一般改動（80% 的情境） |
| **Haiku 4.5** | 最快 | 基礎 | 簡單問答、快速確認、大量批次任務 |

（撰寫本書時的版本，請以 [Anthropic 官方頁面](https://www.anthropic.com/pricing) 為準）

### 我的日常策略

大多數時候用 **Sonnet 4.6**。它夠快、夠聰明，處理 90% 的日常任務沒問題。

遇到以下情境才切 **Opus 4.6**：

- 「這個系統要怎麼設計才能支援未來三年的成長？」
- 「這個 production 只有的 race condition 到底哪裡出問題？」
- 「從 REST 遷移到 GraphQL，幫我規劃完整的 migration path」

**Haiku 4.5** 比較少用，主要是需要跑大量批次查詢，或者對話很短、問題很直接的時候。

一個小秘密：Opus 4.6 有 1M token 的 context window。我的 content-asset-system 專案超大（28KB CLAUDE.md + 幾百個檔案），只有 Opus 能一次吃下整個 context 不遺漏。如果你的專案也很大，這個差距會很明顯。

重點來了：**不用糾結，日常開發直接用 Sonnet 就好。** 遇到感覺 Sonnet 回答得有點淺，再切 Opus，多等 20 秒換一個好答案，很值得。

## 2.4 你的第一次對話：「這個專案在做什麼？」

環境設定好了，現在來開始第一次真正的對話。

我第一次在 content-asset-system 裡開 Claude Code，問它『幫我看一下這個專案在做什麼』。它花了 30 秒掃完所有資料夾、讀了 CLAUDE.md、看了 n8n workflow JSON、甚至找到了 style-dna.yaml，然後跟我說：『這是一個 Git-based 的 YouTube 內容管理系統，用 n8n 做 API 整合，Claude Code 做 AI 生成。』我當下就覺得：這東西懂我。

打開你的任何一個專案（如果沒有現成的，clone 任何一個 GitHub repo 都行），然後在 Claude Code 裡輸入：

```
幫我看一下這個專案的架構，有什麼可以改善的？
```

接下來你會看到 Claude 做這些事：

1. 自動掃描你的專案結構（不是你貼給它的，是它自己去讀）
2. 讀取關鍵檔案：`package.json`、`tsconfig.json`、主要入口點、README
3. 分析程式碼結構、命名規範、依賴狀況
4. 給出具體的改善建議，附上檔案路徑和程式碼位置

這就是 Claude Code 跟 ChatGPT 的本質差異：**它不需要你解釋專案是什麼，它自己會看。**

### 幾個好的起手式問句

第一次用 Claude Code，我推薦從這些問題開始：

```
這個專案的架構是什麼？主要的資料流向怎麼走？
```

```
這個專案有哪些 dependencies 可以考慮升級？
```

```
從 security 角度看，這個 codebase 有什麼需要注意的？
```

這三個問題都能讓你感受到 Claude Code「真的在看你的 code」的感覺，而不只是在聊天。

## 2.5 常見安裝問題排除

### Extension 裝好但登入失敗

確認你的 Anthropic 帳號是 Pro 或以上的方案。Free 方案無法使用 Claude Code Extension。

### 登入後一直轉圈，沒有回應

先檢查網路連線。如果網路沒問題，試試重啟 VS Code 再重新登入。

### 回應很慢，或突然停止

最常遇到的問題：Rate Limit。我每天用 6 小時，大概每 2-3 小時會碰到一次限速。Pro 方案大約 45 則就限速，Max 5x 大約 225 則。碰到的時候，我通常切去另一個 Tab 做不同的事，或者趁機 review code。

可能碰到了 rate limit（用量限制）。Claude Code 的輸入框下方通常會顯示提示訊息。等幾分鐘再試，或考慮升級方案。

### 看不到我的檔案

確認你有**打開資料夾**（File → Open Folder），而不是只打開單一檔案。Claude Code 需要有根目錄才能掃描整個專案。

### Extension 更新後功能跑掉

Anthropic 更新很頻繁。通常重啟 VS Code 就能解決。如果還有問題，嘗試把 Extension 移除再重新安裝。
